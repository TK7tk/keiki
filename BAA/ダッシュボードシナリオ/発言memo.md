# 目的
マーケティング施策のシナリオを具体的アクションに基づいた「接客ジャーニー形式」に詳細化したい。

#依頼ルール
①詳細シナリオは、ユーザーモチベーション、サービスモチベーション、サービス行動/発言、ユーザーモチベーション、ユーザー行動/発言の5項目を追加すること
②サービス行動/発言は①UIタイプ②コンテンツの2つの項目から作成すること
③サービス行動/発言後のユーザーモチベーションは【前向き】【後向き】【ふ～ん】の3つの項目から作成すること
④ユーザーモチベーションが【ふ～ん】となる場合は、次の記載を考えられるものは、サービスモチベーション、サービス行動/発言、ユーザーモチベーション、ユーザー行動/発言として連動していることを明示して
⑤修正後のOUTPUTとは分けて、各々結果が離脱になるものについては離脱を回避する手法の案を出して
⑥.md方式

#OUTPUT形式
①修正後のOUTPUT
②各結果に基づく離脱回避手法案

#修正対象
### シナリオ 0-2：初回でも料金ページまで見たユーザーをAIへ送る（疑問あり）

**Given**

- 初回訪問、検索流入
- 料金ページ到達、滞在>20秒

**When**

1. フローティングバナー（常時）を表示：「疑問をその場で解消」
2. クリックで **Web内AIChat** を起動（AIへ送客）
3. AIはユーザーの初回発話or初回質問項目を受けて、最短で回答→申込導線を提示。又その際に音声入力 & 音声OUTPUTを選択出来る。
    - Pageに基づき初期質問項目がシナリオ方式で設定されている。（シグナルに基づきAIが初期質問項目がCVR精度が高ければAIが初期から実施）
4. AIの意図解釈に基づきAI回答の末尾に「申込Chat or LINEの診断へ」の導線を分ける（質問だけで終わらせない）
5. ユーザーはChat遷移なしで「申込Chat」へ
6. 申込Chatはシナリオ方式（ユーザーシグナルに基づく最適解の出し分け）

**Then**

- 料金/条件の疑問を即時解消し、申込へ滑らせる

**推奨パターン**

- **P3**（意図解釈で会話分岐）＋申込着地はルール（P1）

### シナリオ 3-2：予約完了直後に不安が増え、FAQ/規約を再閲覧 → LINE内AIで不安解消 → 予約維持

**Given（シグナル）**

- Web行動：予約完了後24h以内に「痛み/ダウンタイム/リスク/料金/規約」系ページを複数回閲覧
- LINE：ユーザーが自発的に質問を送る（例：「ダウンタイムどれくらい？」）

**When**

1. LINEに「予約カード」を自動送付（予約日時・場所・持ち物・注意事項・変更導線）
2. 同じスレッド内に「相談（AI）」を常設（自由入力）
3. ユーザーが不安を入力したら、LINE内AIが **短く**回答（長文は避け、箇条書き＋リンク）
4. 回答末尾に固定で「当日の流れ」「注意事項」「変更・キャンセル」導線を提示
5. 変更/キャンセルに関する本文が出た場合のみ、LINE内「手続」フローへ誘導（P1）

**Then**

- 予約キャンセル予備軍をLINE内で回収し、来院率を上げる（No-show/キャンセル抑制）

**推奨パターン**

- LINE内AI相談：**P3**
- 変更/キャンセル手続：**P1**

### シナリオ 3-3：予約後に「メニュー変更/追加施術」に関心→提案を絞ってアップセルに接続

**Given**

- 予約は確定している
- 予約後にユーザーが **別メニューLP / 料金表 / 症例比較 / セット割** を閲覧（=追加購入/変更の仮説）

**When**

1. 予約詳細ページに固定ボタン「施術内容の相談（AI）」を設置
2. 自発入力本文（例：「シミも一緒にやりたい」「予算内で最大効果は？」）をAIが解釈し、選択肢を2案に絞って提示
3. 「予約枠/所要時間/当日可否」のように条件が絡む場合は、AICall（電話）導線を提示（※番号質問はせず"そのまま話せる"入口）
4. 最終確定（変更確定/追加確定）は Webの変更手続（ログが残る）へ着地

**Then**

- 予約を軸に、追加メニュー提案を最小限に絞って追加売上へ

**推奨パターン**

- **P3（提案）＋P1（変更確定手続）＋必要時Call連携**

### シナリオ 3-4：来院前日〜当日にアクセス/地図/注意事項を何度も見る → LINE内AIで即答 → 遅刻/迷子のCallへ自然遷移

**Given（シグナル）**

- Web行動：アクセス、地図、持ち物、注意事項ページの閲覧回数が増える（前日〜当日）
- LINE本文：ユーザーが「場所が分からない」「遅れそう」と入力

**When**

2. ユーザーが自由入力で質問→LINE内AIが回答
3. 「遅れそう」「迷った」など本文が入ったら、LINE内で「今すぐ電話（AICall）」導線を提示
4. 通話後はLINEに戻り「受付済み」「到着予定」「当日変更」などの確認カードを出す

**Then**

- 当日トラブルをLINE中心に処理し、来院率と満足度を上げる

**推奨パターン**

- AI回答：**P3**
- 当日対応/変更：**P1**
- 緊急時：Call
