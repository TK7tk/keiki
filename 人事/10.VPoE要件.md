# VPoE採用要件と戦略

## 1. 現状の課題と背景

### 組織の現状
- **エンジニア組織規模**: 約50名（外国籍メンバー含む）
- **CTOの役割**: 技術面に強みを持つが、組織マネジメントからの解放を希望
- **マネジメント層の不足**: 
  - リーダークラスは存在するが、10名規模をマネジメントできるEMクラスがほぼ不在
  - 現状は5名程度のマネジメントが限界
  - リーダーをマネジメントできる上位層（マネージャー・部長クラス）が不在

### 組織課題の構造
組織課題は大きく2つに分類される:

1. **制度設計・戦略レイヤーの課題**
   - 組織理念やMVV（ミッション・ビジョン・バリュー）が未整備
   - 評価制度などの組織制度設計が不十分
   - 実務のマネジメントでカバーしている状態

2. **実行レイヤーの課題**（より緊急度が高い）
   - マネージャークラスの人材がほぼいない
   - ピープルマネジメント（PL）とプロジェクトマネジメント（PM）の両方が不足
   - 組織全体での課題発見・解決の推進力が弱い

## 2. VPoEの要件

### 【最優先要件】実行力・現場マネジメント能力

現状、**制度設計よりも実行力を優先**することが合意されています。理由は、現場のボトルネック解消が最も緊急であり、制度設計に特化した人材を採用しても短期的な効果が望めないためです。

| 要件項目 | 項目詳細説明 |
| :--- | :--- |
| **50名規模の組織統括** | ・外国籍メンバーを含む約50名のエンジニア組織全体のマネジメントを完全に引き受けられる<br>・多様な背景を持つメンバーに対応できるマネジメントスタイル<br>・CTOが技術面に専念できるよう、組織マネジメントを完全移譲できる |
| **EMクラス人材の育成・強化** | ・10名規模のチームをマネジメントできる人材を育成できる<br>・リーダー層（現在5名程度を見ている層）をマネジメントし、EM層へと引き上げられる<br>・既存メンバーの育成を通じて、マネージャークラスの層を厚くできる |
| **ピープル&プロジェクトマネジメント** | ・ピープルマネジメント（育成、モチベーション、パフォーマンス管理）<br>・プロジェクトマネジメント（複数チームレベルでの推進）<br>・両方が現在不足しており、どちらも対応できることが必須 |
| **組織課題の自律的発見・解決** | ・指示待ちではなく、自ら組織課題を発見できる<br>・チーム内に留まらず、複数チーム・組織全体レベルでの課題に対処<br>・発見から解決までを自律的に推進できる<br>※現状：指示されれば推進できるが、自ら発見することができない |

### 【重要要件】視点・思考力

| 要件項目 | 項目詳細説明 |
| :--- | :--- |
| **経営視点** | ・マネージャークラスには「半分経営視点」が求められる<br>・事業と開発組織をリンクさせ、ビジネス成果に貢献できる<br>・ビジネスサイドとの連携・調整ができる<br>・**戦略は組織に従うのか、組織は戦略に従うのか**のバランスを取れる |
| **中長期的な時間軸** | ・**半年〜1年以上のスパンで組織戦略を描ける**（現状は3ヶ月が限界）<br>・短期・中期・長期のバランスを取りながら目標をコントロールできる<br>・チームをどう強化していくかの中長期ビジョンを持てる |

### 【必要要件】技術面・知識

| 要件項目 | 項目詳細説明 |
| :--- | :--- |
| **技術力** | ・**ミドルクラス以上の技術力が必須**（シニアに近ければ尚良い）<br>・エンジニアとの技術的な会話を理解できる<br>・古い知識に固執せず、現在の技術トレンドを理解している<br>・**3年以上の実務経験**（広めに取る場合。5年程度が理想）<br>・ただし直近5年手を動かしていなくても、それまでの経験の質次第では可 |
| **複雑なシステム経験** | ・決済システムの作り直し、インフラ移行など高難度の経験があると望ましい<br>・現在の組織が抱える複雑な課題と同様の経験 |
| **マネジメント知識** | ・チーム運営・組織運営の体系的知識<br>・行動経済学、心理学など関連分野の知識もあると尚良い<br>・現状、このレベルの知識を持つ人材は組織内にほぼいない |
| **組織成長の実績** | ・**20〜50名規模のエンジニア組織を成長させた経験**<br>・VPoEの下で働いていた経験（VPoE自体の経験である必要はない） |

### 【望ましい要件】制度設計・戦略立案能力

※実行力強化の後に取り組むべき領域として認識

| 要件項目 | 項目詳細説明 |
| :--- | :--- |
| **組織理念・制度設計** | ・MVV（ミッション・ビジョン・バリュー）の策定<br>・評価制度、育成制度などの設計<br>・組織の上位概念を設計し、浸透させられる |

### 参考人物像

**吉澤さん（ビジネスサイドのマネージャー）にエンジニアスキルセットが加わった人材**
- 経営視点を持っている
- 組織をどう動かすかを理解している
- メンバーのレベルと現状のギャップを把握し、適切に前進させられる
- 複数の立場・視点を理解してコミュニケーションできる

## 3. 採用戦略

### 基本方針

| 戦略 | 内容 | メリット・デメリット |
| :--- | :--- | :--- |
| **戦略A:<br>VPoE候補として採用<br>（推奨）** | ・報酬レンジ: 1,500万円〜2,000万円<br>・入社時は「VP候補」として、CTOの下で実績を積む<br>・1.5〜2年かけてVPoE体制を構築 | **メリット:**<br>・パラシュート採用による組織の混乱を回避<br>・既存メンバーの反発を抑えられる<br>・組織適応期間を確保できる<br><br>**デメリット:**<br>・即効性は期待できない<br>・候補者の期待値調整が必要 |
| **戦略B:<br>シニアEM複数採用** | ・報酬レンジ: 900万円 × 2名<br>・実行側を強化し、その中からVPoEに引き上げる | **メリット:**<br>・実行力の即座の強化<br>・リスク分散<br><br>**デメリット:**<br>・VPoE層の不在は継続<br>・既存メンバー育成には時間がかかる（5年程度） |

### 重要な認識

1. **市場の現実**
   - 該当人材は極めて少ない（レバレジーズ経由で3ヶ月に1人来ればいい方）
   - 「宝くじ」と認識し、とにかく母集団を増やす必要がある
   - ウルトラレアな人材が出現した場合は、予算を柔軟に調整して即決する

2. **タイムライン**
   - **入社後1.5〜2年でVPoE体制がワークする**想定
   - 3ヶ月〜半年での劇的な変化は期待しない
   - 短期的な成果を求めるなら、複数シニアEM採用が現実的

3. **VPoEは「育てる」ものではない**
   - VP・役員クラスになる人は、育成されてなる人ではなく、自らその素質を持つ人
   - ミドル〜シニアまでは育成の再現性があるが、それ以上は偶発的
   - だからこそ外部採用が必須

### ターゲット人材

| ターゲット層 | 特徴 | 留意点 |
| :--- | :--- | :--- |
| **メガベンチャーのEM** | ・20〜50名規模の組織成長経験<br>・VPoEの下で実行を担っていた | ・メガベンチャーの方が環境が整っている<br>・当社の方がマネジメント難易度が高い可能性<br>・職務領域が広くなることへの適応力を確認 |
| **成長企業のシニアEM** | ・VPoEではないが、幅広い責任を担っていた<br>・カオスな環境での実績がある | ・制度が整っていない環境への適応力が重要<br>・上流課題がある中でも現場を機能させられるか |

### 募集設計

- **ポジション名**: VPoEとEMの両方で募集（間口を広げる）
- **予算設定**: 上限を高めに設定（最大2,000万円も視野）し、実際は柔軟に調整
- **期待値調整**: 入社時は「VP候補」であり、実績を見て判断することを明示
- **エージェント対応**: ハイクラスエージェントに正確な要件を伝え、長期的な関係構築

### 優先対象プロダクト

採用した人材が最初に見るべきプロダクトの優先順位:

1. **EFO** - 開発能力向上が必要
2. **AI** - マネジメントできる人材がほぼいない（宇佐美さん、ジョバイルさんは IC・リーダークラス志向）
3. **コア** - Kazuo Suzukiさんが単独で回せている状況

### ミーティングで合意すべき内容

1. **採用要件の共有**
   - 実行力重視 vs 制度設計重視の優先順位
   - 完璧な人材は存在しない現実
   - 「VP候補」として採用する戦略

2. **タイムライン期待値の調整**
   ```
   入社 → 半年（組織適応）→ 1年（実績構築）→ 1.5〜2年（VPoE体制確立）
   ```
   - 短期的な劇的変化は期待しない
   - 中長期での組織強化として位置づける

3. **採用戦略の合意**
   - VPoE候補（1名、高報酬）vs シニアEM複数（実行力強化）
   - 予算の柔軟性（上限高く、実際は調整）
   - 募集の間口を広げる（VPoEとEMの両方）

4. **内部育成との比較**
   - 内部育成には5年かかる
   - 事業スピードを考えると外部採用が妥当
   - ただし既存メンバーの育成も並行して継続

## 5. 重要な留意点

### タイムラインに関する留意点

| 項目 | 詳細 |
| :--- | :--- |
| **現実的な期待値** | ・VPoE体制が機能するまで1.5〜2年<br>・最初の半年〜1年は組織理解・適応期間<br>・「3ヶ月で変わる」といった短期思考は危険 |
| **段階的な効果** | ・即効性を求めるなら→複数シニアEM採用<br>・中長期の組織力強化なら→VPoE候補採用<br>・どちらも一定の時間が必要 |

### 採用・オンボーディングに関する留意点

| 項目 | 詳細 |
| :--- | :--- |
| **パラシュート採用リスク** | ・外部から高いポジションで入れると組織が揉める<br>・「VP候補」として入社し、CTOの下で実績を積む設計が重要<br>・既存メンバーとの関係構築期間を確保 |
| **経験の「質」重視** | ・年数だけでなく、どういう経験をしたか<br>・複雑な課題（決済、インフラ移行など）への対応経験<br>・書類ベースでの慎重な見極めが必要 |
| **メガベンチャー出身者の注意点** | ・整った環境からの移行になる<br>・当社の方がマネジメント難易度が高い<br>・制度未整備、職務領域の広さへの適応力を確認 |

### 組織運営に関する留意点

| 項目 | 詳細 |
| :--- | :--- |
| **上流課題との切り分け** | ・事業方向性の変更などはVPoEレベルでは解決できない<br>・経営層での課題とVPoE採用での解決範囲を明確化<br>・VPoEに過度な期待をしない（「そんな人材どこにおるねん」問題） |
| **既存メンバーの育成継続** | ・外部採用だけに頼らない<br>・実行力のある人材に既存メンバーを育ててもらう<br>・マネージャークラスの層を厚くする中長期戦略 |
| **柔軟性の確保** | ・予算は高めに設定しておき、実際は柔軟に調整<br>・ウルトラレアな人材が出現したら即決する覚悟<br>・「宝くじ」なので、とにかく回し続ける |

### 期待値調整に関する留意点

| 項目 | 詳細 |
| :--- | :--- |
| **完璧な人材は存在しない** | ・制度設計と実行力の両方を高レベルで持つ人材は極めて稀<br>・優先順位（実行力）を明確にし、そこで妥協しない<br>・他の要素は育成や他メンバーとの補完で対応 |
| **VPoEは育成対象ではない** | ・VP・役員クラスは育成されてなるものではない<br>・自らその素質を持つ人を見極める<br>・だからこそ採用が極めて重要かつ困難 |
| **予算と効果のバランス** | ・高報酬=即効性ではない<br>・投資として1.5〜2年のリターンを見込む<br>・短期的ROIを求めるなら別の戦略を検討 |

## 補足: 盛山様の例から学ぶこと

盛山様は入社3日目からカジュアル面談を実施するなど、極めて特殊な環境で5年間経験を積んできました。しかし、**当時の盛山様が今受けに来ても、このVPoEポジションにはまらない**という認識が示されています。

これは以下を示唆します:
- **経験の長さよりも質が重要**
- **組織の成熟度と求められる能力は異なる**
- **現在の組織フェーズに合った人材を採用する必要がある**

つまり、候補者の過去の実績だけでなく、**当社の現在のフェーズ（50名規模、制度未整備、実行力不足）にフィットするか**を慎重に見極める必要があります。
